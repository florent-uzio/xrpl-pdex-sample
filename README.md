# XRPL Permissioned DEX Sample

End-to-end demonstration of a permissioned DEX trading flow on the XRP Ledger Testnet using credentials, permissioned domains, trustlines, and offer crossing.

## Prerequisites

- **Node.js** >= 18
- Internet access to the XRPL Testnet (`wss://s.altnet.rippletest.net:51233`)

## Getting Started

```bash
npm install
npm start
```

## Accounts

The sample creates and funds 6 accounts on the Testnet faucet:

| Account | Role |
|---|---|
| **Credential Issuer** | Issues `EUROF_Approved` credentials that gate access to the permissioned domain |
| **Sender** | End-user who holds RLUSD and wants to buy EUROF |
| **Market Maker (MM)** | Liquidity provider who holds EUROF and posts sell offers on the order book |
| **Domain Issuer** | Creates the permissioned domain that restricts DEX trading to credentialed participants |
| **RLUSD Issuer** | Token issuer for the RLUSD IOU |
| **EUROF Issuer** | Token issuer for the EUROF IOU |

## Trading Flow

`npm start` runs the following phases in order:

1. **Account Funding** — All 6 wallets are funded in parallel via the Testnet faucet with 10 XRP each.
2. **Credential Lifecycle** — The Credential Issuer creates `EUROF_Approved` credentials for Sender and MM, then each recipient accepts their credential.
3. **Permissioned Domain** — The Domain Issuer creates a permissioned domain referencing the credential issuer and type. Only accounts holding the accepted credential can place offers on this domain.
4. **Trustlines & IOU Funding** — Both token issuers enable `DefaultRipple`. Sender and MM establish trustlines to both issuers. RLUSD Issuer sends 100,000 RLUSD to Sender; EUROF Issuer sends 100,000 EUROF to MM.
5. **Market Maker Offers** — MM places 5 sell-EUROF / buy-RLUSD offers at rates spread around 1.08 (1.079–1.081), all attached to the permissioned domain.
6. **Sender Trade** — Sender places a buy-EUROF / sell-RLUSD offer at 1.082 which crosses the MM's best ask, executing a trade on the permissioned DEX.
